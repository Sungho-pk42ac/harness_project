# ADR-0002 — MVP 엣지 규칙 = 공유 태그

- **Status**: Accepted
- **Date**: 2026-06-01

## Context

노트 간 "관계"를 정의해야 엣지를 그린다. 후보: 공유 태그, 본문 `[[위키링크]]`, 제목 상호 언급, 생성
시각 근접 등. 노트엔 이미 `tags: string[]`이 있다(tag 기능).

## Decision

MVP 엣지는 **공유 태그**로 한다: 두 활성 노트가 같은 태그를 1개 이상 가지면 무방향 엣지, `weight`=공유
태그 수. 자기 자신 제외, 같은 쌍은 1개.

## Consequences

- 추가 파싱 없이 기존 데이터로 결정적 계산 → 단위 테스트 용이.
- 태그를 공유하지 않는 노트는 고립 노드로 표시(엣지 없음) — 빈약하면 후속에 본문 링크 엣지 추가.

## Alternatives Considered

- **본문 `[[링크]]`/제목 언급**: 파싱·관습 가정 필요, 비결정 위험 → 후속 슬라이스.
- **생성 시각 근접**: 관계 의미가 약함 → 기각.
