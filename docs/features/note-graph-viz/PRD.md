# note-graph-viz — PRD

## 배경

SLNOTE는 노트가 쌓일수록 "어떤 노트들이 서로 관련 있는지"가 보이지 않는다. 태그·검색은 1차원 탐색만
준다. **관계 그래프**로 노트 뭉치(클러스터)와 잊고 있던 연결을 한눈에 보게 한다. Neo4j의 속성 그래프
모델을 개념적으로 참고하되, 별도 그래프 DB 없이 기존 노트에서 클라이언트로 그래프를 만든다.

## 목표 / 비목표

**목표**

- 활성 노트를 노드로, 공유 태그 관계를 엣지로 시각화한다.
- 노드를 클릭해 해당 노트로 바로 이동한다.
- 기존 화면(notes/trash) 흐름을 깨지 않고 graph 뷰를 추가한다.

**비목표(이번 기능 아님)**

- 물리 시뮬레이션 애니메이션, 줌/팬/드래그, 실시간 갱신, 본문 링크/제목 언급 엣지,
  대량 노트 성능 최적화, 외부 graphify HTML 연동, 별도 Neo4j 서버.

## 기능 요구사항 (FR)

| ID   | 우선순위 | 요구사항                            | 수용 기준(요지)                                      |
| ---- | -------- | ----------------------------------- | ---------------------------------------------------- |
| FR-1 | P0       | 노트→그래프 빌드(공유 태그 엣지)    | 같은 태그 가진 두 노트가 엣지로 연결, weight=공유 수 |
| FR-2 | P0       | 그래프 SVG 렌더(원형 배치)          | 노드/엣지가 화면에 보임, 디자인 토큰만               |
| FR-3 | P0       | graph 뷰 토글 + 노드 클릭→노트 열기 | 헤더 토글로 진입, 노드 클릭 시 해당 노트 선택        |
| FR-4 | P1       | 빈 상태                             | 노드 0~1개면 친화 안내                               |

## 성공 지표

- buildNoteGraph·레이아웃 단위 테스트 핵심 분기 전부 그린(결정적).
- 노트 N개에서 공유 태그쌍이 모두 엣지로 나타남(누락 0).
