# 노트 핀 고정 기능 정의서 (확정본)

> 원본 [`spec.md`](spec.md)를 바탕으로 설계 질문의 답변을 반영해 구현 가능한 수준으로 구체화한 문서입니다.
> 태그·검색 기능의 확정본과 동일한 깊이·형식을 따릅니다.

## 기능 개요

중요한 노트에 핀을 달아 노트 목록 **최상단 "고정됨" 섹션**에 분리 표시한다.
검색 없이도 자주 쓰는 소수의 노트에 즉시 닿게 해, 매번 스크롤하는 반복 비용을 없앤다.

## 기능 요구사항

- 노트를 핀으로 고정/해제할 수 있다(토글).
- 고정된 노트는 목록 최상단 "고정됨" 섹션에, 나머지는 그 아래 일반 섹션에 분리 표시한다.
- 핀 상태는 서버에 저장되어 앱을 다시 열어도 유지된다.
- 핀이 하나도 없으면 "고정됨" 섹션은 표시하지 않는다.

---

## 1. 데이터 모델

- 핀 상태는 노트 문서 안에 **불리언 필드**로 임베드한다 → `isPinned: boolean` (ADR-0001).
- 고정되지 않은 노트의 기본값은 **`false`** 로 통일한다.
- `db.json`의 **기존 노트는 수정하지 않는다.** 읽을 때 `isPinned`가 없으면 코드에서 `false`로
  취급한다(`note.isPinned ?? false`). 태그(`note.tags ?? []`)·검색과 동일한 마이그레이션 없는 호환 철학.

```ts
// src/types/note.ts
export interface Note {
  id: string;
  title: string;
  content: string;
  createdAt: string;
  updatedAt: string;
  tags: string[];
  isPinned: boolean; // 추가: 핀 고정 여부 (없으면 false)
}
```

---

## 2. 상태 경계 (서버 데이터 vs 화면 상태)

- 핀 상태는 **노트에 영속되는 서버 데이터**다. 따라서 **`NotesContext`(단일 출처)** 가 소유하고
  변경 함수로 갱신한다. 검색어(`searchQuery`)처럼 화면에만 머무는 UI 상태가 아니다 (ADR-0002).
  - 근거: 핀은 새로고침 후에도 유지돼야 하므로 `selectedNoteId`/`isCreating`/`searchQuery`와 다른 경계다.
    서버 데이터는 Context, 화면 선택 상태는 App이라는 기존 경계를 그대로 지킨다.
- **고정/일반 2섹션 정렬 분기**는 원본 `notes`를 바꾸지 않는 **파생 계산**이다.
  렌더 시점에 순수 함수 `partitionByPinned(notes)`로 두 그룹으로 나눈다(원본 불변, 검색의 `filterNotes`와 동일 철학).

---

## 3. 토글 동작 (입력)

- 핀 토글은 **노트 목록 카드(`NoteItem`)** 에서 한다.
  - 근거: spec의 가치("검색·스크롤 없이 목록에서 즉시 고정/해제")에 가장 부합한다. 편집기를 열지 않고
    목록만 보며 바로 토글할 수 있어야 반복 비용이 줄어든다. (편집기 토글은 이번 범위 제외 — 한 번에 한 기능.)
- 각 카드에 **핀 버튼**(아이콘/이모지)을 둔다. 클릭하면 해당 노트의 `isPinned`가 뒤집힌다.
  - 카드 클릭(=노트 선택)과 충돌하지 않도록 핀 버튼 클릭은 `e.stopPropagation()`으로 선택을 막는다
    (기존 "삭제" 버튼과 동일 패턴).
  - 버튼은 키보드 포커스·클릭 모두 가능해야 한다(접근성). `aria-label`로 현재 동작("고정"/"고정 해제")을 알린다.
- 토글은 **즉시** 서버에 반영한다(편집기 저장 버튼과 무관). 별도 "저장" 단계 없이 클릭 한 번으로 영속화된다.

## 4. 정렬·표시 (매칭·동작)

- 노트 목록을 **두 섹션**으로 나눈다.
  1. **"고정됨" 섹션** — `isPinned === true`인 노트. 목록 최상단.
  2. **일반 섹션** — 나머지 노트. 고정됨 아래.
- **섹션 머리글**: 고정됨 섹션 위에 `고정됨` 라벨(기존 `노트 N개` 라벨과 동일한 작은 캡션 스타일).
  일반 섹션 라벨은 기존 `노트 N개`를 유지하되, 고정 노트를 제외한 **나머지 개수**로 표시한다.
- **각 섹션 내부 정렬**: 두 섹션 모두 **기존 노트 배열 순서를 그대로 유지**한다(거르기/나누기만, 재정렬 없음).
  - 근거: 검색의 "거르기만, 순서 안 바꿈"과 일관. 관련도·최근순 재정렬은 별개 복잡도라 이번 범위 제외.
- **빈 섹션 처리**:
  - 고정된 노트가 **하나도 없으면** "고정됨" 섹션(머리글 포함)을 **표시하지 않는다**.
  - 모든 노트가 고정돼 일반 섹션이 비면 일반 섹션 머리글/목록을 표시하지 않는다.
- **핀 표시(시각 단서)**: 고정된 노트 카드에는 핀 버튼이 **활성 상태**(채워진 아이콘 등)로 보여 한눈에 구분된다.
- **빈 상태/로딩/에러 우선순위**(`NoteList`): `loading` → `error` → 원본 0개("노트가 없습니다")
  → (고정됨 섹션?) + 일반 섹션. 즉 고정/일반 분기는 데이터가 1개 이상 있을 때만 동작한다.

## 5. 저장 흐름 (Context / API)

데이터는 `목록 카드 → NotesContext → api/notes.ts → 서버` 순으로 흐른다. 핀도 이 경로를 따른다.

- **토글 함수 신설(`togglePin`)**: `NotesContext`에 핀 전용 변경 함수를 추가한다 (ADR-0003).
  ```ts
  togglePin(id: string) => Promise<void>
  ```
  내부에서 현재 노트의 `isPinned`를 뒤집어 `api.updateNote(id, { isPinned: next })`를 호출하고,
  응답으로 받은 노트로 로컬 배열을 교체한다(기존 `editNote`의 map 갱신 패턴 재사용).
- **`api/notes.ts` 미수정**: `updateNote(id, { isPinned })`가 이미 `Partial<Note>`를 받으므로
  **시그니처 변경이 필요 없다**. `Note`에 `isPinned`가 추가되면 자동 반영된다(태그와 동일).
- **`addNote` 기본값**: 새 노트는 핀이 꺼진 상태로 시작한다. 호출하는 쪽(편집기/Context)에서
  `isPinned: false`를 채워 보낸다. `api/notes.ts`는 "받은 걸 그대로 보낸다"는 기존 규칙을 유지한다.
- **저장 실패 처리**: 기존 패턴 그대로 **호출부 `try/catch` + `alert`** 로 알린다
  (`NoteList.handleDelete`와 동일). 실패 시 핀 상태는 바뀌지 않는다(응답 기반 갱신).

## 6. 범위 (포함·제외)

### 이번 범위에 포함

- `Note.isPinned: boolean` 필드 추가(기본 `false`, 구버전 `?? false` 방어).
- `NotesContext.togglePin(id)` 변경 함수.
- `NoteList`의 고정/일반 **2섹션 정렬 분기** + "고정됨" 섹션 머리글 + 빈 섹션 미표시.
- `NoteItem`의 **핀 토글 버튼**(접근성·선택 충돌 방지 포함).
- 정렬 분기 순수 함수(`partitionByPinned`) 분리 + 단위 테스트.

### 이번 범위에서 제외 (다음 이슈)

- **핀 개수 제한**(최대 N개 고정).
- **드래그 순서 변경**(고정/일반 내부의 수동 재정렬).
- **편집기(NoteEditor)에서의 핀 토글** — 목록 토글만으로 가치 충족(한 번에 한 기능).
- 핀별 색상·아이콘 커스터마이즈, 고정 노트 접기/펼치기.

## 7. 테스트

- 테스트 도구(Vitest + jsdom + `src/test-setup.ts`)는 갖춰져 있다.
- **`partitionByPinned` 순수 함수 단위 테스트**(핵심):
  - 고정 노트는 `pinned`로, 나머지는 `others`로 분리된다.
  - 두 그룹 모두 원본 순서를 유지한다.
  - `isPinned`가 없는(undefined) 노트는 일반(`others`)으로 취급된다.
  - 고정 노트가 없으면 `pinned`는 빈 배열이다.
  - 원본 배열을 변형하지 않는다(원본 불변).
- **`NoteItem` 핀 버튼 테스트**: 핀 버튼 클릭 시 `onTogglePin(note.id)`가 호출되고,
  카드 선택(`onSelect`)은 호출되지 않는다(stopPropagation).
- (선택) **`NoteList` 섹션 분기 테스트**: 고정 노트가 있으면 "고정됨" 머리글이 보이고,
  없으면 보이지 않는다.
