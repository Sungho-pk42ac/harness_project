## 1. TL;DR

중요한 노트에 **핀을 달아 목록 최상단 "고정됨" 섹션에 분리 표시**하는 기능을 도입한다. 매일 여는 소수의 노트에 **검색·스크롤 없이 즉시 닿게** 한다. 이번 범위는 **핀 토글·2섹션 정렬·영속화**까지이며, **핀 개수 제한·드래그 순서 변경·편집기 토글은 다음 릴리스로 분리**한다.

## 2. 배경 & 문제 (Background & Problem)

- 노트가 늘면서 자주 여는 노트도 목록 안에 섞여 매번 스크롤(또는 검색)해야 닿는다. 단일 사용자가 **매일 여는 노트는 소수**인데, 그 소수에 닿는 비용이 매번 반복된다.
- 태그(v0.1)·검색(v0.2)이 "분류·탐색"을 더했지만, **"자주 쓰는 것을 늘 위에 둔다"** 는 가장 단순한 즉시 접근 수단은 아직 없다.
- **왜 지금**: 핀은 "불리언 하나 + 정렬 분기"라는 가장 작고 확실한 슬라이스다. 세 렌즈(가치·구현 단순성·기존 패턴 정합) 모두 1순위로 꼽았다. 데이터·상태·UI를 얇게 관통해 빠르게 가치를 더한다.

## 3. 목표 & 비목표 (Goals / Non-Goals)

### Goals

- G1. 노트 목록 카드에서 핀을 직관적으로 고정/해제할 수 있다.
- G2. 고정된 노트가 목록 최상단 "고정됨" 섹션에 분리되어, 스크롤 없이 즉시 보인다.
- G3. 핀 상태가 서버에 영속화되어 앱을 다시 열어도 유지된다.

### Non-Goals (이번 릴리스 제외)

- N1. 핀 **개수 제한**(최대 N개 고정).
- N2. 고정 노트 **드래그 순서 변경**(수동 재정렬).
- N3. **편집기(NoteEditor)에서의 핀 토글** — 목록 토글로 충분.
- N4. 핀별 **색상·아이콘 커스터마이즈**, 고정 섹션 접기/펼치기.

## 4. 성공 지표 (Success Metrics)

| 지표        | 정의                                          | 목표                      |
| ----------- | --------------------------------------------- | ------------------------- |
| 분리 정확성 | 고정 노트가 빠짐없이 "고정됨" 섹션에 모이는가 | 단위 테스트 100% 통과     |
| 영속성      | 토글 후 새로고침해도 핀 상태가 유지되는가     | AC로 검증                 |
| 상태 구분   | 고정 0개일 때 "고정됨" 섹션을 숨기는가        | AC로 검증                 |
| 즉시성      | 핀 클릭 후 목록 재배치 체감 지연              | 0 (응답 기반 동기 재렌더) |
| 회귀        | 기존 노트 CRUD·태그·검색 동작 영향            | 회귀 0                    |

> 소규모 학습 프로젝트이므로 정량 트래킹 인프라는 없다. 지표는 **수용 기준(AC)과 단위 테스트·수동 점검**으로 대체한다.

## 5. 사용자 & 유스케이스

- **페르소나**: 노트를 다수 작성·관리하는 단일 사용자(로컬 앱).
- **핵심 유스케이스**
  - U1. 매일 여는 노트(예: `할 일`, `회의록`)에 핀을 달아 늘 위에 둔다.
  - U2. 목록 최상단 "고정됨" 섹션에서 스크롤 없이 바로 그 노트를 연다.
  - U3. 더는 자주 안 쓰는 노트의 핀을 해제해 일반 섹션으로 내린다.

## 6. 기능 요구사항 (Functional Requirements)

우선순위: **P0**=이번 릴리스 필수, **P1**=가능하면 포함.

| ID    | 요구사항                           | 우선순위 | 수용 기준(AC)                                                                   |
| ----- | ---------------------------------- | -------- | ------------------------------------------------------------------------------- |
| FR-1  | 목록 카드에서 핀을 토글한다        | P0       | 카드의 **핀 버튼** 클릭 시 해당 노트의 고정 상태가 뒤집힌다 (ADR-0003)          |
| FR-2  | 핀 상태를 영속화한다               | P0       | 토글 즉시 서버에 반영되어, 새로고침 후에도 상태가 유지된다 (ADR-0001, ADR-0003) |
| FR-3  | 고정 노트를 최상단 섹션에 분리한다 | P0       | `isPinned`인 노트가 목록 맨 위 **"고정됨" 섹션**에 모인다                       |
| FR-4  | 나머지 노트를 일반 섹션에 표시한다 | P0       | 고정되지 않은 노트는 "고정됨" 아래 일반 섹션에 표시된다                         |
| FR-5  | 섹션 내부 순서를 유지한다          | P0       | 두 섹션 모두 **기존 노트 배열 순서를 그대로** 유지한다(재정렬 없음)             |
| FR-6  | 고정 0개면 고정 섹션을 숨긴다      | P0       | 고정된 노트가 없으면 "고정됨" 머리글·섹션이 표시되지 않는다                     |
| FR-7  | 핀 토글과 노트 선택을 분리한다     | P0       | 핀 버튼 클릭은 노트 선택(카드 열기)을 발생시키지 않는다 (`stopPropagation`)     |
| FR-8  | 구버전 노트를 안전 처리한다        | P0       | `isPinned`가 없는 기존 노트는 `?? false`로 일반 섹션에 표시된다 (ADR-0001)      |
| FR-9  | 토글 실패를 사용자에게 알린다      | P0       | 실패 시 기존 패턴(`try/catch` + alert)으로 알리고 상태는 바뀌지 않는다          |
| FR-10 | 고정 노트를 시각적으로 구분한다    | P1       | 고정된 카드의 핀 버튼이 **활성 상태**로 보여 한눈에 구분된다                    |

## 7. 비기능 요구사항 (NFR)

- **아키텍처 정합**: 핀은 **영속 서버 데이터**이므로 `NotesContext`(단일 출처)가 소유하고 `togglePin`으로 갱신한다. 화면 상태(`searchQuery`/`selectedNoteId`)와 섞지 않는다 (ADR-0002).
- **호환성**: 기존 `db.json` 노트는 `isPinned`가 없다. 읽을 때 `note.isPinned ?? false`로 처리해 마이그레이션 없이 동작한다 (ADR-0001). 태그·검색과 동일 철학.
- **성능**: 정렬 분기는 수십~수백 건 동기 `partition`이라 체감 지연 0. 응답 기반 재렌더로 깜빡임 없음.
- **접근성**: 핀 버튼은 키보드 포커스·클릭 가능해야 하며, `aria-label`로 현재 동작("고정"/"고정 해제")을 알린다.
- **스타일**: 시맨틱 토큰(`bg-card`, `border-border`, `text-muted-foreground`, `text-foreground`)만 사용. 임의 hex 색·인라인 style·임의 그림자값 금지(프로젝트 규칙 준수).

## 8. UX / 인터랙션 흐름

```
[핀 토글]
카드의 핀 버튼 클릭 → stopPropagation(선택 막기) → togglePin(id)
   → api.updateNote(id, { isPinned: !isPinned }) → 응답으로 로컬 교체 → 목록 재배치
   (실패 시 alert, 상태 불변)

[목록 표시]
notes → partitionByPinned(notes) → { pinned, others }
   pinned.length > 0 ? "고정됨" 섹션(머리글 + pinned) 표시 : 숨김
   그 아래 일반 섹션(머리글 "노트 N개" + others)
```

빈 상태 분기 우선순위(`NoteList`): `loading` → `error` → 원본 0개("노트가 없습니다") → (고정됨 섹션?) + 일반 섹션.

## 9. 데이터 / 상태 모델 (개요)

- `Note.isPinned: boolean` 추가, 기본값 `false`. 상세 근거는 **[ADR-0001](adr/0001-pin-data-model.md)**.
- 상태 소유 위치(Context vs App): **[ADR-0002](adr/0002-pin-state-ownership.md)**.
- 토글 함수·정렬 분기 방식: **[ADR-0003](adr/0003-toggle-pin-and-partition.md)**.

```ts
export interface Note {
  id: string;
  title: string;
  content: string;
  createdAt: string;
  updatedAt: string;
  tags: string[];
  isPinned: boolean; // 신규
}

// src/utils/partitionByPinned.ts
export function partitionByPinned(notes: Note[]): { pinned: Note[]; others: Note[] };
```

## 13. 의존성

- 태그(v0.1)·검색(v0.2): `Note` 타입을 공유. `isPinned` 추가는 기존 필드와 독립이라 충돌 없음.
- `NotesContext`의 `notes`(단일 출처)와 `updateNote`(`Partial<Note>`) API — 핀 토글은 이를 재사용한다.
- 저장소: JSON Server(`db.json`) — 스키마 자유로워 `isPinned` 추가에 서버 변경 불필요.
