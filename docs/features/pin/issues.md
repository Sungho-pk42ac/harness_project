# 핀 고정 기능 이슈 분해 (Vertical Slicing)

[`PRD.md`](PRD.md)를 기반으로 한 구현 이슈 목록입니다.

## 분해 원칙 — 수직 슬라이싱

각 이슈는 **데이터 → API/Context → UI를 관통하는, 그 자체로 동작하고 사용자에게 보이는 한 조각**입니다.
PRD의 마일스톤(M1~M3)은 참고하되, "사용자가 끝까지 쓸 수 있는 단위"로 재구성했습니다.

- 먼저 **walking skeleton**(PIN-1: 목록 카드에서 핀을 토글하면 서버에 영속화된다)으로 전 계층을 얇게 관통한 뒤,
- 2섹션 정렬 분리(PIN-2) → 빈 섹션·시각 구분·단위 테스트 견고화(PIN-3) 순으로 가치를 덧붙입니다.

## 각 이슈의 구성 — AC와 DoD 구분

- **인수 기준(AC, Acceptance Criteria)** = "**올바르게 동작**하는가?" 기능별 행위 기준. **Given/When/Then**으로 표현.
- **완료 정의(DoD, Definition of Done)** = "**정말 끝났는가?**" 품질·프로세스 게이트. 행위를 다시 적지 않고 완료 조건만.

> 행위(무엇이 맞아야 하나)는 AC에서, 품질(무엇이 갖춰져야 끝인가)은 DoD에서 다룹니다. 둘은 겹치지 않습니다.

## 이슈 ↔ 요구사항(FR) 매핑

| 이슈                           | 담는 FR                      | 의존  |
| ------------------------------ | ---------------------------- | ----- |
| PIN-1 핀 토글 & 영속화         | FR-1, FR-2, FR-7, FR-8, FR-9 | —     |
| PIN-2 고정/일반 2섹션 분리     | FR-3, FR-4, FR-5             | PIN-1 |
| PIN-3 빈 섹션·시각 구분·테스트 | FR-6, FR-10                  | PIN-2 |

> PIN-1이 모든 이슈의 선행입니다. PIN-2는 PIN-1의 데이터·토글 위에서 표시를 나누고, PIN-3은 PIN-2의 섹션을 다듬습니다.

## PIN-1 — 목록 카드에서 핀을 토글하면 서버에 영속화된다

**라벨**: `feature/pin` `P0` `slice:happy-path`
**의존**: 없음 (선행 이슈)

### 설명

핀 기능의 **walking skeleton**. 사용자가 노트 목록 카드의 핀 버튼을 눌러 고정/해제하면, 즉시 서버에 반영되어 새로고침 후에도 상태가 유지된다. 이 한 슬라이스로 `Note` 타입 → `api`/`Context`(`togglePin`) → `NoteItem` 버튼까지 전 계층을 얇게 관통한다. (2섹션 분리는 PIN-2, 빈 섹션·시각 구분은 PIN-3)

### 범위

- `src/types/note.ts`: `isPinned: boolean` 추가.
- 읽기 호환: 기존 노트는 `note.isPinned ?? false`로 처리 (ADR-0001).
- `NotesContext.togglePin(id)` 추가: `api.updateNote(id, { isPinned: !current })` 호출 후 응답으로 로컬 배열 map 교체 (ADR-0003). `addNote` 호출 시 `isPinned: false` 전달.
- `NoteItem`: 핀 토글 버튼 추가(아이콘 + `aria-label`). 클릭 시 `onTogglePin(note.id)` 호출, `e.stopPropagation()`으로 카드 선택과 분리.
- `NoteList`: `togglePin`을 `useNotes()`로 가져와 `handleTogglePin`(try/catch + alert)으로 `NoteItem`에 배선.

### 인수 기준 (AC)

```
시나리오 A: 노트를 고정한다
  Given 고정되지 않은 노트가 목록에 있다
  When 그 카드의 핀 버튼을 클릭한다
  Then 해당 노트가 고정 상태(isPinned=true)가 되고 핀 버튼이 활성 표시로 바뀐다

시나리오 B: 고정을 해제한다
  Given 이미 고정된 노트가 있다
  When 그 카드의 핀 버튼을 다시 클릭한다
  Then 고정이 해제(isPinned=false)된다

시나리오 C: 영속화
  Given 노트를 고정한 상태다
  When 페이지를 새로고침한다
  Then 그 노트의 고정 상태가 그대로 유지된다

시나리오 D: 핀 토글과 노트 선택 분리
  Given 노트 목록을 보고 있다
  When 카드의 핀 버튼을 클릭한다
  Then 핀만 토글되고 그 노트가 편집기로 열리지(선택되지) 않는다

시나리오 E: 구버전 노트 호환
  Given isPinned 필드가 없는 기존 노트가 있다
  When 그 노트의 핀 버튼을 클릭한다
  Then 에러 없이 고정 상태(true)가 되어 영속화된다

시나리오 F: 토글 실패 알림
  Given 서버가 응답하지 않는다
  When 핀 버튼을 클릭한다
  Then 실패 알림이 표시되고 핀 상태는 바뀌지 않는다
```

### 완료 정의 (DoD)

- [ ] 위 AC 시나리오(A~F)가 모두 통과한다.
- [ ] 디자인 시스템 토큰만 사용(임의 색/그림자/인라인 style 없음), husky 디자인 검사·lint·build 통과.
- [ ] 핀 버튼이 키보드 포커스·클릭 모두 가능하다(접근성, `aria-label` 부여).
- [ ] 기존 노트 CRUD(생성·조회·수정·삭제)·태그·검색에 회귀가 없다.
- [ ] 코드 리뷰 후 머지된다.

## PIN-2 — 고정 노트가 목록 최상단 "고정됨" 섹션에 분리되어 보인다

**라벨**: `feature/pin` `P0` `slice:section-split`
**의존**: PIN-1

### 설명

PIN-1로 저장된 핀 상태를 활용해, 노트 목록을 **고정·일반 두 섹션**으로 나눈다. 고정된 노트는 최상단 "고정됨" 섹션에 모이고, 나머지는 그 아래 일반 섹션에 표시된다. 정렬 분기는 순수 함수로 분리해 단위 테스트한다 (ADR-0003).

### 범위

- `src/utils/partitionByPinned.ts` (신규): `notes`를 `{ pinned, others }`로 나누는 순수 함수. `note.isPinned ?? false` 방어, 각 그룹 원본 순서 유지, 원본 불변.
- `NoteList`: `partitionByPinned(notes)`로 나눠 "고정됨" 섹션 + 일반 섹션을 렌더. 일반 섹션 머리글은 기존 `노트 N개`(고정 제외 개수).

### 인수 기준 (AC)

```
시나리오 A: 고정 노트가 최상단 섹션에 모인다
  Given 일부 노트가 고정되어 있다
  When 노트 목록을 본다
  Then 고정된 노트가 목록 맨 위 "고정됨" 섹션에 함께 보인다

시나리오 B: 나머지는 일반 섹션에
  Given 고정/비고정 노트가 섞여 있다
  When 노트 목록을 본다
  Then 고정되지 않은 노트는 "고정됨" 아래 일반 섹션에 보인다

시나리오 C: 섹션 내부 순서 유지
  Given 고정 노트가 여러 개 있다
  When 노트 목록을 본다
  Then 각 섹션의 노트는 원래 목록 순서를 그대로 유지한다(재정렬 없음)

시나리오 D: 토글 시 즉시 재배치
  Given 일반 섹션의 노트를 보고 있다
  When 그 노트의 핀 버튼을 클릭한다
  Then 그 노트가 "고정됨" 섹션으로 즉시 이동한다
```

### 완료 정의 (DoD)

- [ ] 위 AC 시나리오(A~D)가 모두 통과한다.
- [ ] `partitionByPinned`가 순수 함수로 분리되고, 그에 대한 **단위 테스트가 통과**한다(원본 불변·순서 유지·구버전 방어 포함).
- [ ] 디자인 토큰 사용, husky 검사·lint·build 통과.
- [ ] 목록 렌더 성능·레이아웃에 회귀가 없다.
- [ ] 코드 리뷰 후 머지된다.

## PIN-3 — 빈 고정 섹션을 숨기고 고정 노트를 시각적으로 구분한다

**라벨**: `feature/pin` `P1` `slice:polish`
**의존**: PIN-2

### 설명

2섹션 표시를 다듬는다. 고정된 노트가 하나도 없으면 "고정됨" 섹션(머리글 포함)을 숨겨 빈 헤더가 보이지 않게 하고, 고정된 카드의 핀 버튼을 활성 상태로 표시해 한눈에 구분되게 한다.

### 범위

- `NoteList`: `pinned.length === 0`이면 "고정됨" 섹션·머리글을 렌더하지 않음. (모든 노트가 고정돼 일반이 비면 일반 섹션도 숨김.)
- `NoteItem`: `isPinned`일 때 핀 버튼을 활성 스타일(채워진 아이콘 등, `text-foreground` 토큰)로, 아닐 때 비활성(`text-muted-foreground`)로 표시.

### 인수 기준 (AC)

```
시나리오 A: 고정 0개면 고정 섹션 숨김
  Given 고정된 노트가 하나도 없다
  When 노트 목록을 본다
  Then "고정됨" 머리글과 섹션이 표시되지 않고 일반 목록만 보인다

시나리오 B: 첫 고정 시 섹션 등장
  Given 고정 노트가 없어 "고정됨" 섹션이 없다
  When 한 노트를 고정한다
  Then "고정됨" 섹션이 나타나고 그 노트가 그 안에 보인다

시나리오 C: 고정 카드 시각 구분
  Given 고정된 노트와 일반 노트가 있다
  When 목록을 본다
  Then 고정된 카드의 핀 버튼은 활성 표시, 일반 카드의 핀 버튼은 비활성 표시로 구분된다
```

### 완료 정의 (DoD)

- [ ] 위 AC 시나리오(A~C)가 모두 통과한다.
- [ ] 활성/비활성 표시가 디자인 시스템 토큰만으로 구현된다(임의 색 없음), husky 검사·lint·build 통과.
- [ ] 빈 섹션 분기가 빈 상태 우선순위(loading→error→원본 0개→섹션)와 충돌하지 않는다.
- [ ] 코드 리뷰 후 머지된다.
