# 노트 핀 고정 기능 정의서 (초안)

> 거친 아이디어를 최소한의 형태로 적은 초안입니다. 데이터 형태·UI·정렬 규칙 등 세부 결정은
> 다음 단계([`spec-fixed.md`](spec-fixed.md))의 인터뷰에서 확정합니다.

## 기능 개요

중요한 노트에 **핀(고정)** 을 달아, 노트 목록 **최상단의 고정 섹션**에 분리해 표시한다.
매일 여는 소수의 노트에 검색·스크롤 없이 즉시 닿게 한다.

## 핵심 요구사항

- 노트를 핀으로 고정하거나 고정 해제할 수 있다 (토글).
- 고정된 노트는 목록 **최상단 "고정됨" 섹션**에 모아 보여준다.
- 고정되지 않은 나머지 노트는 그 아래 일반 섹션에 보여준다.
- 핀 상태는 저장되어, 앱을 다시 열어도 유지된다.

## 비고 (제약 / 미정)

- 데이터 형태 미정: `Note`에 `isPinned: boolean`을 둘지, 별도 자료구조로 둘지.
- 토글 위치 미정: 노트 목록 카드에서 토글할지, 편집기에서 토글할지, 둘 다일지.
- 핀 상태 변경의 저장 흐름 미정 (`NotesContext` 변경 함수 신설 여부).
- 두 섹션 각각의 **내부 정렬 기준** 미정.
- 구버전 노트(`isPinned` 필드 없음) 호환 처리 미정.

## 이번 범위에서 다루지 않을 것 (러프)

- 핀 **개수 제한**.
- 고정 노트의 **드래그 순서 변경**(수동 재정렬).
