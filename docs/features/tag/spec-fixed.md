# 태그 기능 정의서 (확정본)

> 원본 `spec.md`를 바탕으로, 20개 설계 질문의 답변을 반영해 구현 가능한 수준으로 구체화한 문서입니다.

## 기능 개요

노트에 태그를 추가하고 관리할 수 있다.

## 기능 요구사항

- 노트에 태그를 추가할 수 있다
- 노트에 추가된 태그를 삭제할 수 있다
- 태그 목록을 노트 상세(편집) 화면에서 확인할 수 있다
- 노트 목록에서도 태그를 간략히 확인할 수 있다

---

## 1. 데이터 모델

- 태그는 **문자열 목록**으로 저장한다 → `tags: string[]`
- 태그가 없는 노트의 기본값은 **빈 목록 `[]`** 으로 통일한다.
- `db.json`의 **기존 노트는 수정하지 않는다.** 읽을 때 `tags`가 없으면 코드에서 빈 목록으로 취급한다(`note.tags ?? []`).

```ts
// src/types/note.ts
export interface Note {
  id: string;
  title: string;
  content: string;
  createdAt: string;
  updatedAt: string;
  tags: string[]; // 추가: 태그 목록 (없으면 [])
}
```

---

## 2. 태그 입력 동작 (편집기 / NoteEditor)

- 입력칸에 한 개씩 적고 **Enter** 또는 **쉼표(`,`)** 로 확정 → **칩(알약 모양)** 으로 추가한다.
- 추가된 각 칩의 오른쪽 **`×` 버튼**으로 개별 삭제한다.
- 한 노트의 태그는 **최대 5개**까지 추가할 수 있다. (초과 시 추가하지 않음)

### 정규화 / 검증 규칙

태그를 추가하기 직전, 아래 순서로 처리한다.

1. **정규화**: 앞뒤 공백을 제거하고(`trim`) 영문은 **소문자로 통일**한다. (한글은 영향 없음)
2. **빈 값 무시**: 정규화 후 글자가 하나도 없으면 추가하지 않는다.
3. **중복 검사**: 이미 목록에 있는 태그면 추가하지 않고, **"이미 있는 태그예요" 알림(alert)** 을 띄운다.
4. **개수 제한**: 이미 5개면 추가하지 않는다.

> 소문자 통일을 함께 적용하므로 `Work`와 `work`는 같은 태그로 취급되어 중복 검사가 올바르게 동작한다.

---

## 3. 태그 표시

- **노트 목록(NoteItem)**: 태그를 **작은 칩**으로 표시한다.
  - 태그가 많아 한 줄을 넘으면 **줄바꿈**으로 모두 표시한다(접지 않음).
  - 태그가 **하나도 없으면 아무것도 표시하지 않는다**(빈 안내문 없음).
- **칩 색상**: 임의 색상값 대신 프로젝트의 **시맨틱 색 토큰**(`bg-muted`, `text-muted-foreground` 등)을 사용한다. (다크 모드 등에서 자동으로 어울림)

---

## 4. 저장 흐름 (Context / API)

데이터는 `편집기 → NotesContext → api/notes.ts → 서버` 순서로 흐른다. 태그도 이 경로를 따른다.

- **생성(addNote)**: 시그니처에 인자를 하나 추가한다.
  ```ts
  addNote(title: string, content: string, tags: string[]) => Promise<void>
  ```
- **수정(editNote)**: 손대지 않는다. 이미 "바뀐 항목만 넘기는" 구조이므로 `tags`를 함께 넘기면 된다.
  ```ts
  editNote(id, { title, content, tags });
  ```
- **기본값 채우는 위치**: **호출하는 쪽(편집기/Context)** 에서 항상 `tags`(기본 `[]`)를 함께 보낸다.
  `api/notes.ts`는 "받은 걸 그대로 보낸다"는 기존 규칙을 유지하고 내부에서 기본값을 채우지 않는다.
- **저장 실패 처리**: 기존 패턴 그대로 **`try/catch` 후 `alert`** 로 알린다. (편집기의 기존 방식과 일관)

## 5. 범위

### 이번 범위에 포함

- 태그 추가 / 삭제 / 표시 (목록·편집기)
- 정규화, 빈 값·중복·개수 제한 검증

### 이번 범위에서 제외 (다음 단계)

- **태그 필터** ("특정 태그를 누르면 그 태그가 달린 노트만 보기")
  → 현재 정의서엔 없으므로 별도 정의서로 분리한다. (한 번에 한 기능)

## 6. 테스트

- 테스트 도구(Vitest)는 갖춰져 있으나 아직 테스트 파일이 0개다.
- **핵심 로직(정규화·중복 방지·개수 제한)** 에 한해 가벼운 단위 테스트를 작성한다.
  입력·출력이 분명해 첫 테스트 예제로 적합하다.
