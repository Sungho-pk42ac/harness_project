# 노트 내보내기 기능 정의서 (초안)

> 거친 아이디어를 최소한의 형태로 적은 ① 초안입니다. 다음 단계(`spec-fixed.md`)에서 인터뷰로 모호함을 제거합니다.

## 기능 개요

현재 노트(또는 전체 노트)를 로컬 파일(`.md` / `.json`)로 다운로드한다.

## 핵심 요구사항

- **단일 노트 내보내기**: 지금 열려 있는 노트 1건을 **마크다운(.md)** 파일로 다운로드한다.
- **전체 백업**: 모든 노트를 **JSON(.json)** 파일 하나로 다운로드한다.
- 파일명에는 **노트 제목 / 날짜**가 들어가 어떤 노트·언제 받은 것인지 알 수 있다.
- 서버·데이터 모델을 바꾸지 않고, **클라이언트에서 Blob 다운로드만**으로 구현한다.

## 가치 (왜)

- 로컬 단일 사용자 앱이라 데이터를 **앱 밖으로 꺼내 백업·공유할 수단이 전무**하다.
- 노트가 쌓일수록 유실 위험이 커지는데, 현재는 `db.json`을 직접 만지는 것 외에 방법이 없다.
- "내보내기" 축을 채우면 이후 가져오기(import)·동기화의 토대가 된다.

## 비고 (제약·미정)

- 마크다운 변환 형식(제목을 `#`으로? 태그를 어디에?)은 아직 미정.
- 파일명 규칙(특수문자가 든 제목, 날짜 포맷)은 미정.
- 전체 백업의 JSON 형태(노트 배열 그대로? 메타데이터 감싸기?)는 미정.
- 내보내기 버튼을 어디에 둘지(편집기 안 / 헤더)는 미정.
- **제외(현 시점 명시)**: PDF·HTML 내보내기, 가져오기(import), 클라우드 업로드.
