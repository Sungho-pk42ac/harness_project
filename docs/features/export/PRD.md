## 1. TL;DR

노트를 **로컬 파일로 내보내는** 기능을 도입한다. 현재 노트는 **마크다운(.md)** 으로, 전체 노트는 **JSON(.json) 백업** 한 파일로 다운로드한다. 파일명에 제목·날짜를 담아 식별성을 준다. 서버·모델 변경 없이 **클라이언트 Blob 다운로드만**으로 구현하며, **PDF·HTML·가져오기·클라우드 업로드는 다음 릴리스로 분리**한다.

## 2. 배경 & 문제 (Background & Problem)

- 이 앱은 로컬 단일 사용자용이며, 데이터는 `db.json`(JSON Server) 안에만 있다. 사용자가 **데이터를 앱 밖으로 꺼낼 수단이 전무**하다 — 백업도, 다른 도구로 공유도 불가능하다.
- 노트가 쌓일수록 유실 위험이 커지는데, 지금은 `db.json` 파일을 직접 만지는 개발자 행위 외에 방법이 없다.
- 데이터는 이미 클라이언트 메모리(`NotesContext.notes`)에 전부 있으므로, 서버 왕복 없이 **브라우저 다운로드만**으로 즉시 내보낼 수 있다.
- **왜 지금**: 태그·검색으로 "찾기" 축이 갖춰진 지금, "꺼내기(내보내기)" 축을 채우면 백업·공유라는 실사용 가치를 가장 적은 비용으로 더할 수 있다. 이후 가져오기의 토대도 된다.

## 3. 목표 & 비목표 (Goals / Non-Goals)

### Goals

- G1. 사용자가 현재 보고 있는 노트를 사람이 읽기 좋은 `.md` 파일로 내려받을 수 있다.
- G2. 사용자가 전체 노트를 손실 없는 `.json` 백업 한 파일로 내려받을 수 있다.
- G3. 받은 파일의 이름만 봐도 어떤 노트/언제 받은 것인지 알 수 있다.
- G4. 기존 데이터·서버·아키텍처에 영향 없이(읽기만) 동작한다.

### Non-Goals (이번 릴리스 제외)

- N1. **PDF·HTML 내보내기** (렌더링/라이브러리 의존 → 별도 기능).
- N2. **가져오기(import)** (파싱·검증·머지 정책이 큰 별도 주제).
- N3. **클라우드 업로드 / 공유 링크**.
- N4. **선택 다중 내보내기·zip 묶음**, 다운로드 **토스트/히스토리**.
- N5. **백업 JSON 메타 래퍼/버전 필드** (import이 없어 불필요, YAGNI).

## 4. 성공 지표 (Success Metrics)

| 지표            | 정의                                                    | 목표                       |
| --------------- | ------------------------------------------------------- | -------------------------- |

| 백업 무손실     | JSON 백업이 모든 노트·필드를 손실 없이 담는가           | 라운드트립 점검(파싱 동일) |

| 회귀            | 기존 노트 CRUD·태그·검색 동작 영향                      | 회귀 0                     |

## 5. 사용자 & 유스케이스

- **페르소나**: 노트를 다수 작성·관리하는 단일 사용자(로컬 앱).
- **핵심 유스케이스**
  - U1. 작성한 노트 하나를 `.md`로 받아 다른 사람에게 보내거나 다른 에디터에서 연다.
  - U2. 주기적으로 전체 노트를 `.json`으로 받아 백업해 둔다.
  - U3. 받은 파일명(제목·날짜)으로 폴더에서 원하는 백업을 빠르게 식별한다.

## 6. 기능 요구사항 (Functional Requirements)

우선순위: **P0**=이번 릴리스 필수, **P1**=가능하면 포함.

| ID    | 요구사항                              | 우선순위 | 수용 기준(AC)                                                                        |
| ----- | ------------------------------------- | -------- | ------------------------------------------------------------------------------------ | ---------------------------------------------- |
| FR-1  | 현재 노트를 `.md`로 내보낸다          | P0       | 편집기 "내보내기" 클릭 시 제목·본문·태그를 담은 `.md` 파일이 다운로드된다 (ADR-0001) |
| FR-2  | 전체 노트를 `.json`으로 백업한다      | P0       | 헤더 "백업" 클릭 시 전체 노트를 담은 `.json` 파일이 다운로드된다 (ADR-0002)          |
| FR-3  | 마크다운 변환 규칙을 따른다           | P0       | 제목은 `#` 헤딩, 본문은 원문 유지, 태그 있을 때만 태그 줄 출력                       |
| FR-4  | 태그 없는/구버전 노트를 안전 처리한다 | P0       | `note.tags ?? []` — 태그 없으면 태그 줄·구분선 생략, 에러 없음                       |

| FR-7  | 내보낼 게 없으면 버튼을 비활성화한다  | P0       | 미저장 새 노트는 md 버튼 비활성, 노트 0건이면 백업 버튼 비활성                       |
| FR-8  | 변환을 순수 함수로 분리한다           | P0       | 변환은 서버·상태 변경 없이 파생 계산만, 단위 테스트 가능 (ADR-0001)                  |
| FR-9  | 다운로드 자원을 정리한다              | P0       | 객체 URL은 클릭 직후 `revokeObjectURL`로 해제 (메모리 누수 없음)                     |
| FR-10 | 내보내기 실패를 사용자에게 알린다     | P1       | Blob/URL 생성 예외 시 기존 패턴(`try/catch` + alert)으로 알림                        |

## 7. 비기능 요구사항 (NFR)

- **아키텍처 정합**: 내보내기는 **읽기·파생**이다. `NotesContext`(서버 데이터 단일 출처)는 손대지 않고, **헤더 전역 액션(백업)은 `App.tsx`가 소유**해 `Layout`에 콜백으로 주입한다(검색어를 App이 소유한 것과 같은 경계). 단일 노트 내보내기는 그 노트를 들고 있는 `NoteEditor`가 수행.
- **순수성/테스트성**: 변환은 부수효과 없는 순수 함수, 다운로드는 얇은 부수효과 헬퍼로 분리(ADR-0001).
- **호환성**: `tags`가 없는 기존 노트도 `note.tags ?? []`로 안전 처리.
- **성능**: 수십~수백 건 동기 직렬화는 체감 지연 0. 스트리밍·청크 불필요.
- **스타일**: 시맨틱 토큰(`bg-muted`, `text-muted-foreground`, `border-border` 등)만 사용. 임의 색/그림자/인라인 style 금지.
- **접근성**: 버튼에 명확한 텍스트 라벨, 키보드 포커스·클릭 가능. 비활성 시 `disabled` 표현.

## 8. UX / 인터랙션 흐름

```
[단일 .md 내보내기]
편집기 "내보내기" 클릭 → 저장된 노트(selectedNote) 확보 → noteToMarkdown(note)
  → buildExportFilename(title, 'md') → downloadTextFile(name, md, 'text/markdown') → 브라우저 저장
  (미저장 새 노트면 버튼 비활성 — 클릭 자체 불가)

[전체 .json 백업]
헤더 "백업" 클릭 → notes 확보 → notesToJson(notes)
  → buildExportFilename('notes-backup', 'json') → downloadTextFile(name, json, 'application/json') → 브라우저 저장
  (노트 0건이면 버튼 비활성)
```

버튼 활성 우선순위: `노트 0건` → 백업 비활성 / `미저장 새 노트` → md 비활성 / 그 외 → 활성.

## 9. 데이터 / 상태 모델 (개요)

- 새 상태·서버 호출·모델 변경 **없음**. 클릭 시점 일회성 부수효과로만 동작.
- 변환·파일명 전략: **[ADR-0001](adr/0001-client-export-pure-fn.md)** — 순수 변환 함수 + 얇은 다운로드 헬퍼, 원본 불변.
- 백업 포맷: **[ADR-0002](adr/0002-backup-json-shape.md)** — 노트 배열을 그대로 직렬화(메타 래퍼 없음).

```ts
// src/lib/export.ts
export function noteToMarkdown(note: Note): string;
export function notesToJson(notes: Note[]): string;
export function buildExportFilename(base: string, ext: 'md' | 'json', date?: Date): string;
// src/lib/download.ts
export function downloadTextFile(filename: string, text: string, mime: string): void;
```

## 13. 의존성

- 태그 기능(v0.1): 마크다운 태그 줄은 `Note.tags`에 의존. 없으면 `?? []`로 방어.
- `NotesContext.notes`(단일 출처) 읽기, `App.tsx`의 헤더 액션 소유 경계.
- 브라우저 `Blob` / `URL.createObjectURL` / anchor download — 별도 라이브러리 없이 표준 API만 사용.
