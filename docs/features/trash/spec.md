# 휴지통(소프트 삭제·복구) 기능 정의서 (초안)

> 거친 요구를 최소한의 형태로 적은 초안입니다. 모호한 결정은 `spec-fixed.md`(인터뷰 확정본)에서 채웁니다.

## 기능 개요

노트 삭제를 **즉시 영구 제거 대신 `deletedAt` 마킹(소프트 삭제)** 으로 바꾼다.
삭제된 노트는 목록에서 사라지지만 **휴지통 화면**에서 다시 볼 수 있고, 거기서 **복원**하거나 **영구 삭제**할 수 있다.

## 핵심 요구사항

- 노트를 삭제하면 DB에서 지우지 않고 `deletedAt` 타임스탬프만 채운다(소프트 삭제).
- 일반 노트 목록에는 `deletedAt`이 없는 노트만 보인다.
- **휴지통 뷰**에서 삭제된 노트만 모아 본다.
- 휴지통의 노트를 **복원**하면 `deletedAt`이 지워지고 일반 목록으로 돌아온다.
- 휴지통의 노트를 **영구 삭제**하면 DB에서 실제로 제거된다(되돌릴 수 없음).

## 비고 / 제약

- 로컬 단일 사용자 앱이라 실수 삭제를 되돌릴 수단이 전혀 없는 게 가장 큰 구멍이다. 이 안전망이 목적.
- 저장소는 JSON Server(`db.json`)로 스키마가 자유롭다 — 필드 추가에 서버 변경 불필요.
- 기존 노트에는 `deletedAt`이 없다(마이그레이션 부담 회피 필요).
- 상태 경계 원칙: 서버 데이터는 `NotesContext`, 화면(휴지통 보기 여부)은 `App`의 `useState`.

## 미정(다음 단계에서 결정) — 인터뷰로 채울 질문 후보

- `deletedAt`의 타입·기본값 표현(`null` vs 필드 부재)?
- 삭제 동작의 함수 이름·시그니처(기존 `removeNote`를 어떻게 바꾸나)?
- 휴지통 화면 상태를 어디에 두나(App vs Context)?
- 복원·영구삭제 후 선택 상태(`selectedNoteId`) 처리?
- 휴지통 진입 UI(헤더 토글 vs 별도 라우트)?

## 이번 범위에서 제외(예고)

- **자동 비우기 스케줄**(N일 지나면 자동 영구삭제) — 별도 기능.
- 다중 선택 일괄 복원/영구삭제, "휴지통 비우기" 일괄 버튼.
- 삭제 취소 토스트(Undo 스낵바).
