# note-assistant-agent — 초안 정의서 (spec)

## 개요

SLNOTE에 **대화형 노트 작성 비서 AI 에이전트**를 더한다. 사용자가 채팅으로 자연어 지시를
내리면, 에이전트가 도구(tool)를 호출해 노트를 **검색·생성·수정·삭제**한다. 결과는 채팅과
노트 목록/에디터에 양방향으로 반영된다.

> 이 spec은 거친 출발점이다. 모호함은 `spec-fixed.md`(멀티에이전트 합의)에서 확정한다.

## 핵심 요구사항 (러프)

- 채팅 UI에서 사용자 입력을 받아 에이전트와 멀티턴 대화한다.
- 에이전트는 **tool-calling**으로 노트 CRUD를 수행한다:
  - `searchNotes` / `listNotes` — 내 노트를 자연어로 찾기·요약 (읽기)
  - `createNote` — 새 노트 생성 (쓰기)
  - `updateNote` — 제목/본문/태그/핀 변경 (쓰기)
  - `deleteNote` — 휴지통으로 보내기(소프트 삭제). **실행 전 사용자 확인 게이트 필요** (비가역 위험)
- 에이전트 오케스트레이션은 **LangGraph**(.js)로 표현한다.
- LLM은 **OpenAI**. API 키는 **브라우저에 노출하지 않는다**.
- 모든 노트 접근은 로그인 사용자 본인 것으로 제한된다(**RLS** 준수).
- UI는 **SLNOTE 디자인 시스템**을 따른다.

## 비고 (제약)

- 이 레포의 TDD 7단계와 **수직 슬라이스** 원칙을 따른다(walking skeleton 먼저).
- 단위/컴포넌트 테스트는 **Vitest(`src/`)**, 사용자 여정은 **Playwright E2E**.
- 데이터 접근은 `src/api`만, 상태는 Context, 순수 로직은 `src/lib`/`src/utils` (기존 아키텍처 규약).
- LLM은 **비결정적** — 결정적으로 테스트하려면 LLM 호출을 모킹 가능한 경계 뒤로 격리해야 한다.
- 미확인 전제: **OpenAI API 키 보유 여부**, **LangGraph.js의 번들 동작**. (spec-fixed/PRD에서 다룸)
