# ADR-0004 — 삭제는 소프트 삭제 + 실행 전 확인 게이트

- **Status**: Accepted
- **Date**: 2026-06-01

## Context

에이전트는 비결정적(LLM)이라 의도치 않은 삭제를 부를 수 있다. 삭제는 가장 비가역적인 동작이다.
사용자 통제와 신뢰가 핵심(User Advocate). 단, SLNOTE의 삭제는 이미 **소프트 삭제**(휴지통)다.

## Decision

`deleteNote` 도구는 호출 즉시 실행하지 않는다. 에이전트 루프가 **`PendingAction`**(대상 + 요약)을 만들고
채팅에 **확인 카드**를 띄운다: "노트 〈제목〉을 휴지통으로 보낼까요? 휴지통에서 복구할 수 있어요.
[확인]/[취소]". 사용자가 **[확인]** 하면 그때 `removeNote(id)` 실행 → 도구 결과를 모델에 회신해 루프 재개.
**[취소]** 하면 도구를 실행하지 않고 그 사실을 모델에 알린다.

확인 게이트가 필요한 도구 집합은 **데이터(설정)로** 표현(`TOOLS_REQUIRING_CONFIRMATION`)해 향후
다른 위험 도구로 확장 가능하게 한다.

## Consequences

- LLM 오작동에도 **사용자 승인 없이는 삭제가 일어나지 않는다**.
- 소프트 삭제라 확인 후에도 휴지통에서 복구 가능 — 이중 안전망.
- 확인 게이트 로직(어떤 도구가 게이트 필요한지, 승인/취소 후 루프 전이)은 순수 함수로 단위 테스트.
- 생성/수정은 게이트 없이 즉시 실행(저위험·되돌리기 가능) — 마찰 최소화.

## Alternatives Considered

- **모든 쓰기 도구에 확인**: 마찰 과다(생성마다 확인은 비서의 가치를 깎음) → 삭제만 게이트.
- **하드 삭제 + 확인**: 복구 불가라 더 위험 → 소프트 삭제 유지.
- **LangGraph `interrupt`로 일시정지**: 브라우저 루프에선 `PendingAction` 상태 전이가 더 단순·테스트 용이.
