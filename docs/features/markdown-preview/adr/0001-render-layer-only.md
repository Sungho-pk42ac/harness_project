# ADR-0001 — 렌더 레이어만 추가(저장 포맷 불변) + 토글 상태 위치

- **상태(Status)**: Accepted
- **날짜**: 2026-05-31
- **관련**: [PRD §9](../PRD.md#9-데이터--상태-모델-개요), [ADR-0002](0002-react-markdown-no-raw-html.md)

## Context

본문을 마크다운으로 보여주려면 두 가지를 정해야 한다.

1. **저장 형태**: 마크다운을 어떻게 저장할 것인가? (원문 plain text vs 파싱된 HTML/AST)
2. **토글 상태 위치**: "편집/미리보기" 보기 상태를 어디에 둘 것인가? (편집기 로컬 / `App` / `NotesContext`)

제약·맥락은 다음과 같다.

- 기존 `Note.content`는 **plain text 문자열**이며, 저장 흐름은 `편집기 → NotesContext → api/notes.ts → 서버`로 단순하다.
- 이 코드베이스는 **상태 경계 원칙**이 명확하다 — 서버 데이터는 `NotesContext`, 화면 선택 상태(`selectedNoteId`, `isCreating`)는 `App`의 `useState`. 폼 로컬 상태(`title`, `content`, `tags`)는 `NoteEditor` 안에 있다.
- 마크다운 렌더는 **읽기 전용 표현**일 뿐, 데이터 변경이 아니다.

## Decision

**렌더 레이어만 추가한다. 저장 포맷은 바꾸지 않는다.**

- `Note.content`는 계속 **마크다운 원문(= plain text)** 으로 저장·전송한다. 타입·DB·API 스키마는 손대지 않는다.
- 마크다운은 **렌더 시점에만** 해석한다(읽기 전용 파생). 저장 경로에는 전혀 개입하지 않는다.

**토글 상태(`view`)는 편집기 로컬 상태에 둔다.**

```ts
// src/components/NoteEditor.tsx
type EditorView = 'edit' | 'preview';
const [view, setView] = useState<EditorView>('edit');
```

- "지금 본문을 어떻게 보고 있나"는 **한 편집기 안에서만 의미 있는 화면 상태**다. 서버 데이터와 무관하므로 `NotesContext`에 두지 않고, 화면 전역에서 공유될 필요가 없으므로 `App`에도 올리지 않는다.
- 기본값은 **`edit`**. 선택 노트가 바뀌면(폼 동기화 `useEffect`) **`edit`로 리셋**한다.
- 미리보기는 **현재 `content` 상태**(미저장 편집 포함)를 렌더한다.

## Consequences

**긍정**

- **위험이 낮다** — 저장 포맷을 건드리지 않으므로 기존 CRUD·태그·검색에 회귀가 없다. 마이그레이션 불필요.
- 토글이 로컬 상태라 **다른 컴포넌트·Context와 결합이 없다**. 노트 전환 시 자연스럽게 리셋된다.
- 미리보기가 순수 파생이라, 끄면 원래 편집 경험이 그대로 남는다(되돌리기 쉬움).

**부정 / 트레이드오프**

- 보기 모드가 **노트별로 기억되지 않는다**(전환하면 항상 편집으로). 현재 단일 편집기 UX에선 오히려 예측 가능. 기억이 필요해지면 별도 ADR로 위치를 재검토한다.
- 저장 형태가 plain text라 **렌더 비용이 매 미리보기마다** 든다. 본문 한 건이라 무시 가능.

## Alternatives Considered

1. **파싱된 HTML을 저장**(`content`를 HTML로 변환해 저장)
   - 기각 이유: 저장 포맷 변경 → 기존 노트 마이그레이션·역호환 부담, XSS 표면 증가, "원문 편집" 단순함 상실. 범위(렌더 레이어만)와 충돌.
2. **토글 상태를 `App`/`NotesContext`로 올리기**
   - 기각 이유: 한 편집기 안에서만 쓰는 화면 상태를 전역으로 끌어올리면 상태 경계가 흐려진다. 공유 요구가 없어 YAGNI.
3. **분할 동시 뷰**(편집+미리보기 나란히)
   - 기각 이유: 스크롤 동기화·레이아웃 복잡도가 커 이번 범위 밖. 토글이 더 단순하고 모바일에도 유리. 다음 슬라이스 후보.
