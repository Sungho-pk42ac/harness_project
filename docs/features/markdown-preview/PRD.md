## 1. TL;DR

노트 편집기에서 본문을 **마크다운으로 렌더**하고 **편집 ↔ 미리보기를 토글**하는 기능을 도입한다. 저장 포맷(plain text)은 그대로 두고 **읽기 전용 렌더 레이어만** 얹는다. 이번 범위는 **헤딩·리스트·강조·코드** 렌더와 토글까지이며, **분할 뷰·이미지·GFM·syntax highlighting·저장 포맷 변경은 다음 릴리스로 분리**한다.

## 2. 배경 & 문제 (Background & Problem)

- 현재 본문은 `<textarea>`의 raw 텍스트로만 보인다. 사용자가 `#`, `-`, `**` 같은 마크다운 표기를 써도 **읽을 때 구조가 드러나지 않아** "읽기 좋은 메모"라는 핵심 가치가 절반만 충족된다.
- 데이터는 이미 `content`에 plain text로 들어 있어, **저장 포맷을 건드리지 않고 렌더 레이어만** 추가하면 위험 없이 가치를 더할 수 있다.
- **왜 지금**: CRUD·태그·검색 기반이 안정적인 지금, "읽기 경험"을 끌어올리는 가장 저비용·고체감 기능이 마크다운 렌더다. 헤딩·리스트·코드만 지원해도 체감이 크다.

## 3. 목표 & 비목표 (Goals / Non-Goals)

### Goals

- G1. 편집기에서 본문을 마크다운으로 렌더한 결과를 읽기 모드로 볼 수 있다.
- G2. 편집 ↔ 미리보기를 토글 한 번으로 전환할 수 있다.
- G3. 헤딩·리스트·강조·코드가 의도대로 렌더된다.
- G4. 저장 포맷은 그대로(plain text)이며 기존 저장 흐름에 회귀가 없다.

### Non-Goals (이번 릴리스 제외)

- N1. **분할 동시 뷰**(에디터+프리뷰 나란히, 스크롤 동기화).
- N2. **이미지 업로드/렌더**.
- N3. **저장 포맷 변경**(마크다운 AST·HTML 저장 등).
- N4. **GFM 확장**(표·체크박스·취소선·자동 링크).
- N5. **코드 syntax highlighting**(색칠).
- N6. **목록(`NoteItem`) 카드 본문 렌더** — 이번엔 편집기 한정.

## 4. 성공 지표 (Success Metrics)

| 지표        | 정의                                           | 목표                      |
| ----------- | ---------------------------------------------- | ------------------------- |
| 렌더 정확성 | 헤딩·리스트·강조·코드가 기대 태그로 렌더되는가 | 컴포넌트 테스트 100% 통과 |
| 토글 정확성 | 편집/미리보기 전환이 정확히 동작하는가         | AC로 검증                 |
| 보안        | raw HTML 입력이 실행되지 않는가(XSS)           | 단위 테스트로 검증        |
| 저장 회귀   | 저장 포맷·CRUD·태그·검색 동작 영향             | 회귀 0 (저장 포맷 불변)   |

> 소규모 학습 프로젝트이므로 정량 트래킹 인프라는 없다. 지표는 **수용 기준(AC)과 단위 테스트·수동 점검**으로 대체한다.

## 5. 사용자 & 유스케이스

- **페르소나**: 노트를 다수 작성·관리하는 단일 사용자(로컬 앱).
- **핵심 유스케이스**
  - U1. 헤딩·리스트로 구조를 잡은 노트를 작성하고, 미리보기로 정돈된 모습을 확인한다.
  - U2. 코드 스니펫을 ` ``` ` 블록으로 적고, 미리보기에서 코드 블록으로 읽는다.
  - U3. 미리보기에서 내용을 확인하다 편집으로 돌아가 수정한다.

## 6. 기능 요구사항 (Functional Requirements)

우선순위: **P0**=이번 릴리스 필수, **P1**=가능하면 포함.

| ID    | 요구사항                                 | 우선순위 | 수용 기준(AC)                                                                     |
| ----- | ---------------------------------------- | -------- | --------------------------------------------------------------------------------- |
| FR-1  | 본문을 마크다운으로 렌더한다             | P0       | `content`가 `react-markdown`으로 렌더되어 미리보기 영역에 표시된다 (ADR-0001)     |
| FR-2  | 편집/미리보기를 토글한다                 | P0       | 토글 버튼으로 본문 영역이 `<textarea>`(편집) ↔ 렌더 결과(미리보기)로 전환된다     |
| FR-3  | 헤딩을 렌더한다                          | P0       | `# 제목`이 `<h1>`(이하 레벨도) 으로 렌더된다                                      |
| FR-4  | 리스트를 렌더한다                        | P0       | `- 항목`은 `<ul><li>`, `1. 항목`은 `<ol><li>`로 렌더된다                          |
| FR-5  | 강조를 렌더한다                          | P0       | `**굵게**`→`<strong>`, `*기울임*`→`<em>`                                          |
| FR-6  | 코드를 렌더한다                          | P0       | 인라인 `` `code` ``→`<code>`, ` ``` ` 블록→`<pre><code>`                          |
| FR-7  | 토글 기본값은 편집이다                   | P0       | 노트를 열거나 새로 만들면 항상 편집 모드로 시작한다                               |
| FR-8  | 노트 전환 시 편집 모드로 리셋한다        | P0       | 선택 노트가 바뀌면 미리보기였더라도 편집 모드로 돌아간다                          |
| FR-9  | 미리보기는 미저장 변경을 즉시 반영한다   | P0       | 저장하지 않은 현재 `content` 편집 내용이 미리보기에 반영된다                      |
| FR-10 | 저장 포맷을 변경하지 않는다              | P0       | 저장 시 서버에는 `content` 원문(plain text)이 그대로 전송된다 (ADR-0001)          |
| FR-11 | raw HTML을 실행하지 않는다               | P0       | 본문의 `<script>`·`<img onerror>` 등 raw HTML이 그대로 실행되지 않는다 (ADR-0002) |
| FR-12 | 미리보기 모드에서도 저장/취소가 동작한다 | P1       | 미리보기 상태에서 저장 버튼을 누르면 정상 저장된다                                |

## 7. 비기능 요구사항 (NFR)

- **아키텍처 정합**: 토글 상태(`view`)는 **편집기 로컬 화면 상태**다(`NoteEditor`의 `useState`). 서버 데이터(`NotesContext`)와 섞지 않는다 (ADR-0001). 미리보기는 데이터 변경 경로에 개입하지 않는 **읽기 전용 파생 렌더**다.
- **보안**: `react-markdown` 기본값(HTML 미허용)을 유지하고 `rehype-raw` 등 raw HTML 허용 플러그인을 쓰지 않는다 (ADR-0002).
- **호환성**: 저장 포맷·타입·DB 스키마 불변. 기존 노트는 그대로 렌더된다(마이그레이션 없음).
- **성능**: 본문 한 건 렌더로 비용이 작다. 토글 시에만 렌더하므로 키 입력마다 비용이 들지 않는다.
- **스타일**: 시맨틱 토큰(`text-foreground`, `text-muted-foreground`, `bg-muted`, `border-border`)만 사용. 임의 색/그림자/인라인 style 금지.
- **접근성**: 토글 버튼은 키보드 포커스·클릭 가능, 현재 모드를 `aria-pressed` 등으로 알린다.

## 8. UX / 인터랙션 흐름

```
[보기 전환]
편집기 열림 → view='edit'(기본) → textarea로 본문 편집
  토글 클릭 → view='preview' → 같은 자리에 react-markdown 렌더(읽기 전용)
  토글 클릭 → view='edit' → 다시 textarea
[노트 전환]
다른 노트 선택/새 노트 → view='edit'로 리셋
[저장]
저장 버튼 → (모드 무관) content 원문 그대로 addNote/editNote → 서버 반영 → 실패 시 alert
```

제목·태그·저장/취소 버튼은 **두 모드에서 항상 동일**하게 보인다. 토글은 **본문 영역만** 바꾼다.

## 9. 데이터 / 상태 모델 (개요)

- 저장 데이터: **변경 없음**. `Note.content`(plain text/markdown 원문) 그대로. 근거는 **[ADR-0001](adr/0001-render-layer-only.md)**.
- 토글 상태 위치: **[ADR-0001](adr/0001-render-layer-only.md)** — `NoteEditor`의 `view: 'edit' | 'preview'`.
- 렌더 보안 정책: **[ADR-0002](adr/0002-react-markdown-no-raw-html.md)** — `react-markdown` + raw HTML 미허용.

```ts
// src/components/NoteEditor.tsx
type EditorView = 'edit' | 'preview';
const [view, setView] = useState<EditorView>('edit');

// src/components/MarkdownPreview.tsx (신규)
interface MarkdownPreviewProps {
  content: string;
}
export function MarkdownPreview({ content }: MarkdownPreviewProps): JSX.Element;
```

## 13. 의존성

- **신규 의존성**: `react-markdown`(런타임). 기존 React 19와 호환되는 버전 사용.
- 디자인 토큰: `text-foreground`/`bg-muted` 등은 `src/index.css`의 CSS 변수에 의존.
- 편집기: `NoteEditor`의 `content` 상태(단일 출처)와 폼 동기화 `useEffect`.
