# 마크다운 미리보기 기능 정의서 (확정본)

> 원본 `spec.md`를 바탕으로, 설계 질문의 답변을 반영해 구현 가능한 수준으로 구체화한 문서입니다.

## 기능 개요

노트 편집기에서 본문(`content`)을 **편집(raw 텍스트)** 과 **미리보기(마크다운 렌더)** 로 전환해 볼 수 있다. 저장 포맷은 그대로 plain text이며, 미리보기는 **읽기 전용 렌더 레이어**다.

## 기능 요구사항

- 편집기에서 본문을 마크다운으로 렌더한 결과를 볼 수 있다.
- 편집 ↔ 미리보기를 **토글 버튼**으로 전환한다.
- 헤딩·리스트·강조·코드(인라인/블록)가 올바르게 렌더된다.
- 저장되는 데이터는 바뀌지 않는다(plain text 원문 그대로).

---

## 1. 데이터 모델

- **데이터 모델 변경 없음.** 본문은 기존 `Note.content: string`(plain text/markdown 원문)을 그대로 사용한다.
- 마크다운은 **렌더 시점에만** 해석한다. 저장·전송 형식은 변하지 않는다.
- 따라서 `src/types/note.ts`, `db.json`, `api/notes.ts`의 스키마는 손대지 않는다.

```ts
// src/types/note.ts — 변경 없음
export interface Note {
  id: string;
  title: string;
  content: string; // markdown 원문(= plain text)으로 그대로 저장
  createdAt: string;
  updatedAt: string;
  tags: string[];
}
```

---

## 2. 상태 모델 (편집/미리보기 토글)

- 토글 상태는 **편집기 로컬 상태**다 → `NoteEditor`의 `useState<'edit' | 'preview'>('edit')`.
  - 근거: "지금 편집기를 어떻게 보고 있나"는 **순수 화면 상태**이며 한 컴포넌트 안에서만 의미가 있다. 서버 데이터(`NotesContext`)와 무관하고, `App`이 소유한 선택/생성 상태(`selectedNoteId`, `isCreating`)처럼 화면 전역에서 공유될 필요도 없다. (상태 경계 원칙: 서버=Context, 화면=App/컴포넌트)
- **기본값은 `edit`** — 노트를 열거나 새로 만들면 항상 편집 모드로 시작한다.
- 선택 노트가 바뀌면(`selectedNoteId`/`isCreating` 변경) 토글을 **`edit`로 리셋**한다. (기존 폼 동기화 `useEffect`에 함께 둔다)

```ts
// src/components/NoteEditor.tsx
type EditorView = 'edit' | 'preview';
const [view, setView] = useState<EditorView>('edit');
```

## 3. 입력 / 동작 (토글)

- 편집기 상단(또는 본문 영역 근처)에 **편집 | 미리보기 토글 버튼**을 둔다.
- **편집 모드**: 기존 `<textarea>` 그대로 — 본문을 자유롭게 수정한다.
- **미리보기 모드**: `<textarea>` 대신 **렌더된 마크다운**(읽기 전용)을 같은 자리에 보여준다.
- 토글은 **본문 영역만** 전환한다. 제목 입력, 태그 영역, 저장/취소 버튼은 두 모드에서 **항상 동일하게** 보인다.
- 미리보기 모드에서도 **저장/취소는 가능**하다(본문은 마지막 편집 내용 그대로 저장).
- 미리보기는 **현재 편집 중인 `content` 상태**를 렌더한다(저장 전 미저장 변경도 즉시 반영).

---

## 4. 매칭 / 렌더 규칙 (지원 문법)

`react-markdown`으로 렌더한다. **이번 범위에서 보장**하는 문법:

| 문법          | 입력 예시              | 렌더 결과          |
| ------------- | ---------------------- | ------------------ |
| 헤딩          | `# 제목`, `## 소제목`  | `<h1>`, `<h2>` …   |
| 순서 리스트   | `1. 첫째`              | `<ol><li>`         |
| 비순서 리스트 | `- 항목`, `* 항목`     | `<ul><li>`         |
| 강조          | `**굵게**`, `*기울임*` | `<strong>`, `<em>` |
| 인라인 코드   | `` `code` ``           | `<code>`           |
| 코드 블록     | ` ```\ncode\n``` `     | `<pre><code>`      |

- **보안**: raw HTML은 렌더하지 않는다(`react-markdown` 기본값 = HTML 미허용). XSS 방어를 위해 `rehype-raw` 같은 raw HTML 허용 플러그인을 **쓰지 않는다**. (ADR-0002)
- **GFM(표·체크박스·취소선 등)** 은 이번 범위 밖이다. 필요 시 다음 슬라이스에서 `remark-gfm`을 추가한다.
- **코드 syntax highlighting**(색칠)은 범위 밖이다. 코드 블록은 단색 `<pre><code>`까지만.

---

## 5. 표시 (스타일)

- 렌더 영역은 **시맨틱 디자인 토큰**만 사용한다(`text-foreground`, `text-muted-foreground`, `bg-muted`, `border-border` 등). 임의 색/그림자/인라인 style 금지.
- 마크다운 기본 태그(`h1`, `ul`, `code` 등)에 **최소한의 타이포 간격**을 입혀 읽기 좋게 한다(굵기·여백·코드 배경). Tailwind 유틸리티로 처리한다.
- 미리보기 영역 높이는 편집 `<textarea>`와 **시각적으로 유사**하게(레이아웃 점프 최소화).
- 빈 본문을 미리보기하면 빈 영역(또는 옅은 안내)으로 보인다 — 에러 없이.

---

## 6. 저장 흐름

- **저장 흐름 변경 없음.** 저장은 기존과 동일하게 `content` 원문(plain text)을 그대로 보낸다.
  - 생성: `addNote(title, content, tags)`
  - 수정: `editNote(id, { title, content, tags })`
- 미리보기는 **렌더 전용**이라 저장 경로(`Context → api/notes.ts → 서버`)에 전혀 개입하지 않는다.
- 저장 실패 처리도 기존 패턴(`try/catch` + `alert`) 그대로.

---

## 7. 범위

### 이번 범위에 포함

- `react-markdown` 의존성 추가.
- 편집기 본문 영역의 **편집 / 미리보기 토글**(편집기 로컬 상태).
- **헤딩·리스트·강조·코드(인라인/블록)** 렌더.
- 렌더 영역의 디자인 토큰 스타일링.

### 이번 범위에서 제외 (다음 단계)

- **분할 동시 뷰**(에디터+프리뷰 나란히 스크롤 동기화).
- **이미지 업로드/렌더**.
- **저장 포맷 변경**(마크다운 AST 저장 등).
- **GFM 확장**(표·체크박스·취소선·자동 링크) — 필요 시 `remark-gfm` 별도 슬라이스.
- **코드 syntax highlighting**(색칠).
- 목록(`NoteItem`) 카드의 본문 마크다운 렌더 — 이번엔 편집기 한정.

---

## 8. 테스트

- 본문 렌더는 **DOM 결과가 분명**하므로 컴포넌트 단위 테스트가 적합하다(`@testing-library/react`).
- 핵심 테스트 포인트:
  - 마크다운 입력 → 기대 태그(`<h1>`, `<ul><li>`, `<strong>`, `<code>`)로 렌더되는가.
  - 토글이 편집(`<textarea>`)과 미리보기(렌더 결과)를 올바르게 전환하는가.
  - 미리보기 모드에서도 저장/취소가 동작하는가.
  - raw HTML 입력이 **그대로 실행되지 않는가**(보안).
