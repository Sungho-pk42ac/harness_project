# SLNOTE 디자인 시스템

만들고 싶은 노트 웹 **SLNOTE**의 디자인 캡쳐를 기반으로 정리한 디자인 가이드입니다.
앞으로 UI를 구현할 때 이 문서를 일관된 기준으로 삼습니다.

> 이 문서는 "현재 앱 상태"가 아니라 **목표 디자인(target design)** 을 규정합니다.
> 기존 토큰(`src/index.css`)을 출발점으로 삼되, 인디고 포인트 컬러를 더한 전체 시스템을 정의합니다.

## 브랜드 한 줄 정의

**밝은 배경의 지식베이스/위키형 노트 플랫폼.** 콘텐츠(글·카드)가 주인공이고, 인디고는 거들 뿐.

## 디자인 원칙

1. **콘텐츠 우선** — 연한 회색 배경 위에 화이트 카드를 띄워 콘텐츠를 부각한다. 장식보다 가독성.
2. **포인트는 인디고로만** — 인디고(primary)는 핵심 액션(주요 버튼·FAB·링크 강조)에만 쓴다. 면적이 아니라 점으로.
3. **넉넉한 여백** — 요소를 빽빽이 채우지 않는다. 여백이 위계를 만든다.
4. **시맨틱 토큰 사용** — 임의 색상값·매직 그림자값 대신 정의된 토큰(`bg-primary`, `text-muted-foreground` 등)을 쓴다. (프로젝트 전역 컨벤션과 동일)

## 참고

- 스타일링 스택: **Tailwind CSS v4** (`@tailwindcss/vite`, 별도 config 없음). 색·폰트·radius는 `src/index.css`의 `@theme` CSS 변수로 정의.
- 토큰을 코드에 적용하려면 [colors.md](colors.md)의 "적용용 CSS" 블록을 `src/index.css`에 붙여넣으면 됩니다.
